Common questions

Why not just use Auth0 or Clerk out of the box?
You can! But there's still implementation work: integrating with your app, customizing the UI, setting up SSO with specific providers, configuring RBAC for your use case, and handling edge cases. This package handles all of that. You get a production-ready auth system, not just an auth provider account.
Is 3 days really enough for SSO?
Yes, if you use a proper auth platform (Supabase, Auth0, etc.) and know what you're doing. The heavy lifting is done by the platform; I handle the integration, configuration, and testing. If you need custom SAML configuration for a specific enterprise customer, that might take an extra day or two.
What if I need to pass a security audit?
The implementation follows security best practices: hashed passwords, rate limiting, audit logs, secure session management. This covers most SOC2 and ISO 27001 requirements for authentication. If you need specific compliance features (e.g., password rotation policies, IP whitelisting), let me know upfront and I'll include them.
Do you include the frontend components?
Yes. You get working login, registration, password reset, and 2FA setup forms. Styled to match your app (I'll use your design system or Tailwind defaults). Mobile-responsive and accessible. Not just backend APIs — complete, ready-to-use auth flows.
What tech stacks do you support?
React/Next.js frontend, Node.js or Python backend. Database-agnostic (works with PostgreSQL, Supabase, MongoDB, etc.). If you're using something else, let's discuss — I can usually adapt.
